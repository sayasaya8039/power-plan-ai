# note記事作成ルール

## 作成タイミング
- Git操作完了後、ユーザーに確認して作成
- 記事は `articles/` フォルダに保存
- GitHubにはアップロードしない（.gitignoreに追加）
- 記事はmarkdown記法で作成

## リサーチ（必須）
- 海外サイト、Reddit、Web、ブログ、SNS、YouTube
- 参考URL・動画があれば記事内に埋め込み

## 文章構造

### 1. 導入（150〜250字）
- 読者の悩みを具体的に代弁
- 自身の体験を1文入れる
- 情景描写や擬態語を使用

### 2. 本論（1,000〜1,500字）
- 小見出しは感情語＋具体名詞
- 順序をランダムに入れ替える
- 数字は「取得方法→計算式→結果」をセット

### 3. 結論（200〜400字）

## 文体ガイドライン
- 一文の長さ：5〜120字でばらつかせる
- 語尾：「です」「でしょう」「ます」を循環（同一語尾2連続まで）
- 接続詞を多様化（とはいえ／それでも／ふと／さて）
- **絵文字を積極的に使用**
- 専門用語は分かりやすい日本語で説明
- 読者への問いかけを挿入（最大3個）

## SEO対策
- Google検索1位・上位表示を目指す
- キーワード密度は1.5%以下
- PREP法やSDS法を固定せず段落順を変える

## 必須要素

### 記事末尾に挿入（タグの前）
```
皆様の意見はどうでしょうか？
良かったらコメントで教えて下さい。
フォロー＆スキもお願いします♪

この記事への感想やご質問、お仕事のご依頼など、
お気軽にメッセージをお送りください♪
📩メッセージはこちらから
https://note.com/alvis8039/message
```

### ハッシュタグ
- 記事の一番最後にスペース区切りで20個
- note、Google検索1位を狙えるタグ

## 画像プロンプト（Midjourney用）
- 4枚程度、記事内に挿入位置を明記
- 英文で作成
- 舞台：日本、日本人
- スタイル：フォトリアル、レイトレーシング、16:9横長
- 記事末尾に一覧をリスト化

## 注意事項
- 表はnoteで崩れるため箇条書きに変換
- **性別は記事に明記しない**
- AIスコア30%未満を目指す
