# ドキュメント記述ルール

## 基本方針
- **日本語**で記述する
- 簡潔で分かりやすい文章を心がける
- 技術用語は必要に応じて補足説明を加える

## 見出し構造
- 見出しは明確な階層で始める（h1 → h2 → h3）
- 見出しだけで内容が把握できるようにする

## コードブロック
- コード例には必ず言語を指定する
- 実行可能なサンプルコードを提供
- 出力結果も併記する

```python
# 例: ユーザー情報の取得
user = get_user(user_id=123)
print(user.name)
# 出力: "山田太郎"
```

## 画像・図表
- 画像には必ずalt属性を付ける
- 図表には説明文（キャプション）を追加
- スクリーンショットは最新の状態を維持

## リンク
- 外部リンクは有効性を確認
- 内部リンクは相対パスを使用
- リンク切れがないか定期的にチェック

## 更新履歴
- 重要な変更はCHANGELOGに記録
- 日付形式は `YYYY-MM-DD` を使用
