# claude-mem プラグイン活用ガイドライン

## 概要

claude-memは過去の会話・作業履歴を記憶し、長期プロジェクトでの文脈維持を支援するプラグインです。

---

## 使用タイミング

### 必須で使用する場面

| タイミング | 操作 |
|----------|------|
| 作業開始時 | 過去の発言・作業工程を確認 |
| 重要な決定時 | 設計方針・技術選定を記憶に保存 |
| エラー解決後 | 解決策を記録して再発防止 |
| セッション終了時 | 進捗状況と次回TODOを保存 |

### 積極的に使用する場面

- 長期プロジェクトでの作業継続時
- 複数セッションにまたがるタスク
- ユーザーの好み・スタイルの把握
- プロジェクト固有の用語・略語の記録

---

## 記憶に保存すべき情報

### 必須保存

```
- プロジェクト名と概要
- 技術スタック（使用言語、フレームワーク、ライブラリ）
- 重要な設計決定とその理由
- 発生したエラーと解決策
- ユーザーの明示的な指示・好み
```

### 推奨保存

```
- ファイル構造の概要
- API設計・データモデル
- 環境設定（デプロイ先、認証方式等）
- 進行中のタスクと完了状況
- 次回セッションでのTODO
```

---

## 使用例

### 作業開始時

```
1. claude-memで過去の記憶を確認
2. 前回の作業内容・進捗を把握
3. 未完了タスクがあれば継続
4. 新規タスクなら文脈を踏まえて開始
```

### 重要な決定時

```
例: 「認証方式をFirebase Authに決定」
→ 記憶に保存:
  - 決定事項: Firebase Auth使用
  - 理由: Googleログイン必須、デモモード対応のため
  - 関連ファイル: src/lib/firebase/config.ts
```

### エラー解決後

```
例: 「ドラッグ&ドロップが動かない問題を解決」
→ 記憶に保存:
  - 問題: dnd-kitでカラム間移動ができない
  - 原因: closestCorners衝突検出が空カラムで失敗
  - 解決: pointerWithin + rectIntersectionに変更
```

## Memory MCP との連携

claude-memプラグインとMemory MCPサーバーを併用する場合：

| 用途 | 使用ツール |
|------|----------|
| セッション内の短期記憶 | claude-mem |
| プロジェクト横断の長期記憶 | Memory MCP |
| ユーザー固有の設定・好み | 両方 |

---

## 注意事項

- **機密情報（APIキー、パスワード等）は記憶に保存しない**
- 記憶の確認は作業開始時に1回で済ませる（過度な確認はコスト増）
- 古い情報は適宜更新・削除して最新状態を維持
- 記憶に依存しすぎず、必要に応じてコードを直接確認
